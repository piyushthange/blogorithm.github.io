What is KERNEL?

Have you ever wondered how and editor works or how the game starts and why does the game has a fancy Loading... bar at the beginning? Why you need a big chunkier graphics card to play games? The answer to this is the Operating System which is synonymously used for KERNEL. The main function of the kernel is to interact with the hardware(RAM, CPU, Storage Devices, Keyboard) and provides an execution platform to application programs so that the user can be at concinience and do his work effectively without worrying about the resources available. Now if you want to play a game and you clik the icon of game so as to open it. To do this the game has to request the kernel to use hardware resources like monitor, keyborad, mouse, GPU & whatever applies to play. Then the kernel evaluates the request and check wheather the permissions game is asking are fine or is there an illegal access of memory or if the hardware isn't available. If anything related to it is improper it'll send some signal to the monitor that there's an issue with hardware or memory access or so on and stop it then there. Hence in modern OS's the user programs cannot play with the low-level hardware of the computer system so as to prevent a crash of working and system and never retrieving it back.

Who are involved in the development of the Linux Kernel?

We'll just look at the history of Linux in just a snap. This project was started by Linus Torvalds projecting a simple email (see it here) on the internet and getting people onboard for contribution and he a benevolent dictator till date. The Linux Kernel is being developed by Hobbyists around the globe over the internet using mails(sending, replying, reviewing, agreeing & disagreeing).
Now that the kernel is defined the thing to note in here is that the Linux kernel is open sourced licensed under GPL v2.0 i.e anyone personally can change it, use it and dustribute it for free. If the changes are good then it should be sent to upstream and get the changes accepted by the maintainer of particular subsystem and then the maintainers forward changes to Linus so as the changes get merged in kernel. In my sense there are three levels of people in development
  1. Linus Torvalds - Maintainer of Linux Kernel
  2. Subsytem Maintainers - Subsystem can have one or more maintainers.
  3. Contributors/Developers - Send fixes in the kernel and be the part of development.
