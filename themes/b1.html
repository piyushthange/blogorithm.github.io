What is KERNEL?

Have you ever wondered how and editor works or how the game starts and why does the game has a fancy Loading... bar at the beginning? Why you need a big chunkier graphics card to play games? The answer to this is the Operating System which is synonymously used for KERNEL. The main function of the kernel is to interact with the hardware(RAM, CPU, Storage Devices, Keyboard) and provides an execution platform to application programs so that the user can be at concinience and do his work effectively without worrying about the resources available. Now if you want to play a game and you click the icon of game so as to open it. To do this the game has to request the kernel to use hardware resources like monitor, keyborad, mouse, GPU & whatever applies to play. Then the kernel evaluates the request and check wheather the permissions game is asking are fine or is there an illegal access of memory or if the hardware isn't available. If anything related to it is improper it'll send some signal to the monitor that there's an issue with hardware or memory access or so on and stop it right there. Hence in modern OS's the user programs cannot play with the low-level hardware of the computer system so as to prevent a crash of working system and never retrieving it back. Ok now we have know what kernel is now let's look what's the history of kernel & where it has evolved from.

Behind The Scenes of Linux Kernel

We'll just look at the history of Linux in just a snap. Linux is based on Unix but it's not Unix. Unix was only used in the universities & was created at AT&T Bell labs by Ken Thompson, Dennis Ritchie and other team members. Linux was started by Linus Torvalds as a hobby project & launched a simple email (see it here)on the internet which lead to one of the biggest open project on internet. People got onboard for providing their contributions to linux where Linus Torvalds became a benevolent dictator & still continues to be one. The Linux Kernel is being developed by Hobbyists, company heired employees to contribute to specific subsystems, engineers and many more around the globe over the internet using mails (sending, acknowledging, reviewing, agreeing & disagreeing).
Now that the kernel is defined the thing to note in here is that the Linux kernel is open sourced licensed under GPL v2.0 i.e anyone personally can change it, use it and dustribute it for free. If the changes are good then it should be sent to upstream and get the changes accepted by the maintainer of particular subsystem and then the maintainers forward these changes to Linus so as the changes get merged in kernel. There are three levels of people in development
  1. Linus Torvalds - Creator & Maintainer of Linux Kernel. Accept patches from Subsystem Maintainers
  2. Subsytem Maintainers - Subsystem(wifi, usb, ipc, mm) can have one or more maintainers.
  3. Contributors/Developers - Send fixes in the kernel after testing to a particular assigned maintainer or developer.
Linux is open sourced i.e the source code of linux is free. Anyone can edit, use & redistribute the code. Linux Kernel is Licensed under GPL-2.0(General Public License V-2.0). There could be multiple Licenses as per a particular source file. These are available in COPYING File(Described further). Read the GPL-2.0
