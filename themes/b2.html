Let's get started with a new session with understanding some new term's like Mail client, Patch, Release Cycle, Cscope, ctags, git, ncurses, vim(Text editor), dwarves,etc. The kernel development takes place on mailing lists. Make changes in the kernel test them create a patch send them to respective maintainer & mailing list. Get the patch reviewed & accepted or rejected that's it. A patch is a change made in the kernel source & this can be send through mail. To create a patch git is the tool in hand for us. we can use git format-patch -1 After formatting patches you can check who's exactly the mainter of that subsystem using scripts/get_maintainer.pl name.patch. Cscope is a finding tool To check if the patch has coding style correct or not you can use scripts/checkpatch.pl filename
